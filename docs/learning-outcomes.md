# Learning Outcomes

This project helped in understanding several important cloud and serverless computing concepts.

---

# Concepts Learned

## Serverless Architecture

Understood how backend applications can run without managing traditional servers by using AWS Lambda.

---

## API Gateway Routing

Learned how API Gateway receives HTTP requests and routes them to backend Lambda functions.

---

## Lambda-Based Backend Execution

Understood how Lambda functions execute backend logic dynamically only when triggered.

---

## CRUD Operations using HTTP Methods

Learned how different HTTP methods are used for CRUD operations.

```text id="kvowt0"
POST   → Create
GET    → Read
PUT    → Update
DELETE → Delete
```

---

## Single Lambda Handling Multiple Operations

Understood how a single Lambda function can process multiple CRUD operations by checking the incoming HTTP method from API Gateway.

---

## DynamoDB Integration

Learned how AWS Lambda connects with DynamoDB using Boto3 for NoSQL database operations.

---

## NoSQL Database Design

Understood:

* partition keys
* item structure
* flexible schema design
* serverless database concepts

---

## IAM-Based Access Control

Learned how IAM roles securely allow Lambda functions to access DynamoDB resources.

---

## Monitoring & Observability

Understood how:

* CloudWatch stores Lambda logs
* SNS can be used for monitoring alerts

---

## Difference Between Traditional & Serverless Backends

Traditional backend systems use continuously running servers.

Serverless systems execute backend logic only when requests are received.

---

# Overall Outcome

This project provided practical understanding of:

* Modern serverless backend architecture
* API-driven cloud applications
* AWS service integration
* Event-driven workflows
* Production-style cloud design concepts
